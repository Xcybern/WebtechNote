Web 技术栈与客户端 / 服务器模型

前端 · Browser

客户端（你看到和点到的）

① HTML · 结构骨架 标题/段落/列表
② CSS · 外观样式 颜色/布局/动画
③ JavaScript · 交互行为 按钮点击/表单校验
  • 运行在 浏览器沙箱 内，安全隔离，不直接碰数据库/硬盘
  • 通过 HTTP 向服务器 发出请求，拿回 HTML/CSS/JS 或 JSON
  • 能在本地存 localStorage / cookie，记住你的选择
发出请求 ➜
后端 · Server

服务器（负责处理与“做决定”）

Web 程序
Node / Python / Java / Go…
数据层
数据库 / 缓存 / 文件
静态资源
HTML / CSS / JS / 图片
  • 接收请求 → 鉴权/业务逻辑/读写数据 → 返回 响应
  • 常见返回：网页内容 或 JSON 接口（给前端渲染）
  • 会做 安全校验、速率限制、日志监控
⬅ 返回响应

一句话理解：浏览器 负责显示与交互，服务器 负责处理与存储，它们用 HTTP 说话。

资源寻址（URI / URL / URN）

地址本身 · Human-readable

URL 是“找东西的完整地址”

schemehttps://
hostwww.example.com
path/docs/index.html
query?q=html
fragment#top
示例 URL https://cs.st-andrews.ac.uk/courses/cs5063?week=1#html
scheme=https host=cs.st-andrews.ac.uk path=/courses/cs5063 query=?week=1 fragment=#html
  • URI 是“标识符”的总称；URL 能定位资源；URN 更像“不变的名字”。
  • scheme 决定怎么访问（如 https、ftp）。
  • host 是域名；path 是站内路径；query 是参数；fragment 跳到页面某处。
解析与路由 · Machine-routable

DNS 与 IP：把“文字地址”变成“数字地址”

DNS 解析器
浏览器/操作系统先问这里
DNS 服务器
从根域 → 顶级域（.com） → 权威服务器逐级查询
IP 地址
IPv4 例：93.184.216.34 IPv6 例：2606:2800:220::
  1. 浏览器拿到 URL → 先用 DNS 把 域名 解析为 IP。
  2. 拿到 IP → 用 TCP/UDP 发包（常见：HTTPS 走 TCP:443）。
  3. 服务器收到 → 返回内容（HTML/CSS/JS 或 JSON）。
  • IPv4 32 位（地址较短、资源紧张）；IPv6 128 位（地址巨多、改进路由与扩展性）。
  • 同一域名可同时有 A（IPv4）和 AAAA（IPv6）记录；系统会优先选择能连通、策略更优的那个。

说明：IPv6 并不是某个国家发明；是由 IETF（互联网工程任务组） 在 1990 年代中期制定的新一代互联网协议， 1995 年首次发布（RFC 1883），1998 年标准化（RFC 2460），2017 年更新为现行标准（RFC 8200）。

记忆法：URL 像门牌地址，DNS 像电话簿，IP 像电话号码；先查电话簿（DNS），拿到号码（IP），再打电话（网络请求）。

HTTP：请求与响应（浏览器怎么和服务器说话）

Request · 我想要点啥

HTTP 请求（从浏览器发出去）

方法 GET POST PUT DELETE
目标 URL（/users/123?tab=profile）
请求头（Headers）
  • Host：域名（例：example.com）
  • Accept：我能接收的类型（HTML / JSON / 图片）
  • Authorization：登录凭证 / 令牌
  • Cookie：会话信息
请求体（Body）（可选）
例如：JSON、表单、文件
发出请求 ➜
Response · 给你结果

HTTP 响应（服务器回过来）

状态码 200 OK 401/403 404 Not Found 500 Server Error 301/302
类型 HTML JSON CSS / JS 图片
响应头（Headers）
  • Content-Type：内容类型（如 text/html、application/json）
  • Set-Cookie：让浏览器保存会话信息
  • Cache-Control：缓存策略
  • Location：重定向目标（配合 301/302）
响应体（Body）
例如：网页 HTML 或 JSON 数据
⬅ 返回响应
HTTPS 在 TCP 之上工作， 把 请求 和 响应 安全地送达两端。

快速记忆：浏览器 提出诉求（请求），服务器 给出答案（响应）。请求里写“想干啥”和“我是谁”；响应里写“结果如何”和“给你的内容”。

源码

文档结构（Document Tree）

源码（最小页面骨架）
<!doctype html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>Hello</title>
  </head>
  <body>
    <h1>First Page</h1>
    <p>This is my first page.</p>
  </body>
</html>
效果预览（仅渲染 body 内容）

First Page

This is my first page.

DOCTYPE 声明 HTML5；lang 指定语言； <head> 放元数据与外链；<body> 放可见内容。
